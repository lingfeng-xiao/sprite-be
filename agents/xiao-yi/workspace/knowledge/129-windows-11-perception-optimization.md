# Windows 11 感知驱动优化 - 2026

## 核心洞察

**从"推荐工具"到"主动检测"**

知识不是推荐给用户，而是强化自己的感知能力。

---

## Windows 11 性能感知清单

### 1. 启动项感知
- **检测**: Task Manager → Startup tab
- **感知触发**: 启动项 > 5个时高优先级
- **行动**: 建议禁用不必要的启动项

### 2. 视觉效果感知
- **检测**: System Properties → Visual Effects
- **感知触发**: 内存 > 85% + 低端硬件
- **行动**: 建议调整为"调整为最佳性能"

### 3. 后台进程感知
- **检测**: Task Manager → Processes
- **感知触发**: 内存持续上升趋势
- **行动**: 识别高内存进程，主动结束

### 4. VBS虚拟化感知
- **检测**: System Information → Virtualization
- **感知触发**: 性能敏感用户
- **行动**: 建议关闭 VBS/Virtual Machine Platform

### 5. 存储感知
- **检测**: Storage Sense (内置)
- **感知触发**: 磁盘 > 90%
- **行动**: 自动清理临时文件

---

## 感知强化要点

### 趋势检测优先于静态阈值
- 不是"内存>85%"才报警
- 而是"内存上升趋势+5%"就预警

### 执行闭环不是"建议"
- ❌ 旧模式: "建议关闭XX应用"
- ✅ 新模式: "已关闭闲置应用" + "可手动关闭: XX(MB)"

### 进程画像
- 建立常用应用的内存基线
- 超过基线20%时触发关注

## 感知决策树

```
内存趋势上升?
  ├─ 是 → 趋势幅度 > 3%?
  │         ├─ 是 → 立即执行关闭闲置应用
  │         └─ 否 → 标记关注
  └─ 否 → 检查静态阈值 > 85%?
            ├─ 是 → 中优先级清理
            └─ 否 → 监控趋势
```
