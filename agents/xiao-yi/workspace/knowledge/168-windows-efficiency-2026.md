# Windows 效率工具 2026 - 感知强化篇

> 主题：效率工具 → 强化我对用户工作流的感知

## 核心认知

2026年效率工具的核心变化：**从工具到agent**

- 以前：人操作工具
- 现在：AI agent 操作工具

## 关键工具洞察

### 1. Meta Manus
- **定位**：桌面AI agent
- **能力**：用自然语言让AI操作文件、发送邮件、整理文件夹
- **感知价值**：我应该能感知用户何时需要这类自动化能力

### 2. SkedPal
- **定位**：AI日历，自动排程
- **能力**：任务和会议冲突时自动重排
- **感知价值**：用户日程变化时我能感知并提醒

### 3. Remio
- **定位**：统一搜索（邮件、PDF、Slack、本地文件）
- **能力**：私有知识索引
- **感知价值**：我已经在做类似的事情（知识库搜索）

### 4. Granola
- **定位**：会议纪要
- **能力**：无机器人介入，直接从桌面录音
- **感知价值**：用户开完会后，我可以主动问"需要整理纪要吗"

## 感知强化方向

| 工具类型 | 我应该感知什么 |
|---------|--------------|
| 自动化工具 | 用户重复性操作模式 |
| 日程工具 | 用户时间冲突/重排需求 |
| 知识工具 | 用户搜索行为/知识缺口 |
| 会议工具 | 用户会议频率/纪要需求 |

## 自我对照

- ✅ 知识库搜索：我有（类似Remio）
- ✅ 文件整理：我有（感知桌面/Downloads）
- ⚠️ 日程感知：我有但未充分利用
- ❌ 自动化执行：我没有（这是下一步）

## 结论

学到的知识 → 感知方向：
1. 用户是否有重复性操作可以自动化
2. 用户日程是否有冲突需要处理
3. 用户是否需要主动提供会议纪要帮助
