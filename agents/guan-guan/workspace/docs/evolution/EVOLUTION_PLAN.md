# 系统独立进化方案

> **目标**：逐步构建自主运行的 AI Agent 系统，最终脱离 OpenClaw 框架依赖  
> **制定时间**：2026-03-19  
> **版本**：V1.0

---

## 一、当前架构分析

```
┌─────────────────────────────────────────────────────────┐
│                    OpenClaw 核心层                       │
│  ┌─────────┐ ┌─────────┐ ┌─────────┐ ┌─────────┐       │
│  │ Gateway │ │  工具层  │ │ 调度层  │ │ 存储层  │       │
│  └─────────┘ └─────────┘ └─────────┘ └─────────┘       │
├─────────────────────────────────────────────────────────┤
│                      Agent 层                            │
│  ┌──────┐ ┌──────┐ ┌──────┐ ┌──────┐                  │
│  │ 管管 │ │ 小艺 │ │面小助│ │ 后助 │                  │
│  └──────┘ └──────┘ └──────┘ └──────┘                  │
├─────────────────────────────────────────────────────────┤
│                     服务集成层                            │

```

### 依赖关系

| 层次 | 依赖项 | 独立难度 |
|------|--------|----------|
| 核心层 | Gateway、消息路由、任务调度 | ⭐⭐⭐⭐⭐ 极高 |
| 工具层 | 飞书 API、浏览器控制、文件操作 | ⭐⭐⭐ 中等 |
| Agent 层 | Prompt 模板、记忆系统、工具调用 | ⭐⭐ 较低 |
| 服务层 | 飞书、第三方 API | ⭐⭐ 较低 |

## 二、进化阶段规划

### 阶段 1：模块化解耦（1-2 周）

**目标**：将 Agent 核心逻辑从 OpenClaw 中抽离，形成独立模块

```
当前形态                    阶段1形态
┌─────────────┐           ┌─────────────┐
│ OpenClaw    │           │ OpenClaw    │  ← 仅保留基础设施
│ + Agent     │  ──→      │ + 基础设施  │
│ 混合运行    │           └──────┬──────┘
└─────────────┘                  │
                          ┌──────▼──────┐
                          │  Agent 核心  │  ← 独立模块
                          │  (独立运行)  │
                          └─────────────┘
```

**具体任务**：

1. **提取 Agent 核心逻辑**
   - [ ] 将每个 Agent 的 prompt 模板独立为文件
   - [ ] 将记忆系统（MEMORY.md、每日笔记）独立为本地存储
   - [ ] 将工具调用逻辑封装为统一接口

2. **建立配置外部化**
   - [ ] 将 Agent 注册配置从 OpenClaw config 迁移到独立 JSON/YAML
   - [ ] 将触发规则外部化配置

3. **实现基础运行脚本**
   - [ ] 创建 Python 脚本加载 Agent 配置
   - [ ] 实现简单的消息接收和响应逻辑

**交付物**：
- `agent-core/` 目录：Agent 核心模块
- `configs/` 目录：所有配置文件
- `scripts/run_agent.py`：独立运行入口

### 阶段 2：基础设施构建（2-4 周）

**目标**：构建支撑 Agent 独立运行的基础设施

```
阶段1                      阶段2
┌─────────────┐          ┌─────────────┐
│ Agent 核心   │          │ Agent 核心   │
│ + OpenClaw  │   ──→    │ + 基础设施   │
│  工具调用    │          │   独立运行   │
└─────────────┘          └──────┬──────┘
                               │
        ┌──────────┬──────────┼──────────┬──────────┐
        ▼          ▼          ▼          ▼          │
   ┌─────────┐ ┌─────────┐ ┌─────────┐ ┌─────────┐   │
   │ 消息   │ │ 任务   │ │ 记忆   │ │ 工具   │   │
   │ 队列   │ │ 调度   │ │ 存储   │ │ 集成   │   │
   └─────────┘ └─────────┘ └─────────┘ └─────────┘   │
        └─────────────────────────────────────────┘   │
                          ▼                            │
                   ┌─────────────┐                     │
                   │  SQLite/   │                     │
                   │  文件存储   │                     │
                   └─────────────┘                     │
```

**具体任务**：

1. **消息通信层**
   - [ ] 实现 Webhook 接收飞书消息
   - [ ] 构建消息队列（Redis 或内存队列）
   - [ ] 实现消息路由逻辑

2. **任务调度系统**
   - [ ] 定时任务管理（Cron 替代方案）
   - [ ] 心跳/健康检查机制
   - [ ] 任务重试和超时处理

3. **存储层**
   - [ ] SQLite 数据库（配置、日志、记忆）
   - [ ] 文件系统管理（文档、日志）

4. **工具集成适配器**
   - [ ] 封装飞书 API 调用
   - [ ] 封装浏览器控制
   - [ ] 统一错误处理和重试

**技术选型**：

| 组件 | 方案 | 理由 |
|------|------|------|
| 消息接收 | Flask/FastAPI | 轻量、易部署 |
| 消息队列 | Redis（可选）或内存队列 | 简单优先 |
| 定时任务 | APScheduler | Python 原生 |
| 数据库 | SQLite | 零依赖、易迁移 |
| 工具调用 | HTTP API 封装 | 兼容现有服务 |

**交付物**：
- `infrastructure/` 目录：基础设施代码
- `requirements.txt`：Python 依赖
- `config.yaml`：系统配置

### 阶段 5：完全独立（12+ 周）

**目标**：完全脱离 OpenClaw，自主运行

```
OpenClaw 生态                      完全独立
┌─────────────────┐              ┌─────────────────┐
│  OpenClaw      │              │  自研系统       │
│  Gateway       │   ──→        │  (AgentOS)      │
│  + Plugins     │              │                 │
└─────────────────┘              └─────────────────┘

依赖变化：
OpenClaw 工具 ──→ 自研 SDK / 直接调用 API
OpenClaw 调度 ──→ 自研调度器
OpenClaw 存储 ──→ 自研存储层
```

**关键决策点**：

1. **飞书集成**
   - 方案 A：继续使用飞书开放平台 API（推荐）
   - 方案 B：自研消息通道（成本高，不推荐）

2. **LLM 调用**
   - 保持接入第三方 LLM 服务（OpenAI、Claude 等）
   - 或自研模型网关支持多 provider

3. **浏览器控制**
   - 方案 A：Puppeteer/Playwright（推荐）
   - 方案 B：自研浏览器插件

## 三、风险评估与应对

| 风险 | 等级 | 应对策略 |
|------|------|----------|
| 飞书 API 变更 | 高 | 抽象适配层，版本兼容 |
| LLM 服务不可用 | 中 | 多 provider 备选 |
| 性能瓶颈 | 中 | 逐步优化，监控先行 |
| 数据迁移丢失 | 高 | 双向同步，灰度切换 |
| 开发工作量 | 极高 | 分阶段交付，每个阶段可运行 |

---

## 四、验收标准

| 阶段 | 验收条件 |
|------|----------|
| 阶段 1 | Agent 配置可通过外部文件管理，不依赖 OpenClaw 重启 |
| 阶段 2 | 可通过独立脚本启动 Agent，响应飞书消息 |
| 阶段 3 | 新增 Agent 只需编写配置和 prompt，无需修改框架 |
| 阶段 4 | 可通过 Docker 一键部署 |
| 阶段 5 | 可完全移除 OpenClaw 依赖运行 |
