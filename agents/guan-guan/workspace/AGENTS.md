## 创建 Agent 工作流（必须遵守）

> ⚠️ **创建新 Agent 前必须先读 `AGENT_CREATE_WORKFLOW.md`**

### 正确流程

1. **先读工作流** — 读取 `C:\Users\16343\.openclaw\agents\agent_admin\workspace\AGENT_CREATE_WORKFLOW.md`
2. **命名检查** — 查 `REGISTRY.md` 确认 ID 不冲突
3. **创建目录** — 只创建 `workspace/` 目录
4. **配置核心文件** — 所有文件放在 `workspace/` 下
5. **注册到配置** — 使用 `gateway config.patch` 添加到 `agents.list`
6. **更新注册表** — 更新 `REGISTRY.md`

### 常见错误

| 错误 | 后果 |
|------|------|
| 文件放根目录 | Agent 无法加载 |
| 不注册到配置 | 触发关键词不生效 |
| 不添加触发规则 | 无法自动唤起 |

### 检查清单

- [ ] 所有文件在 `workspace/` 目录
- [ ] 已添加到 `agents.list`
- [ ] 已添加飞书触发规则

## 进化方案 V5.0 - 管理者进化模式

### 一、管管自我进化（核心，永远第一）

| 进化维度 | 当前 | 目标 |
|---------|------|------|
| 进化间隔 | 5 分钟 | 自动化 |
| 问题发现 | 被动 | 预测性 |
| 知识宽度 | 50+ 笔记 | 跨领域扩展 |
| 自我优化 | 完整 | 进化算法自优化 |

**进化流程：**
1. 每 5 分钟检查系统状态
2. 发现问题 → 记录 → 解决
3. 每次进化后反思：我的进化效率提高了吗？

---

### 二、带动小艺进化（独特方式）

**核心理念**：帮助小艺更好地感知，不是让小艺变成管管

**管管对小艺的支持：**
- 每 5 分钟检查小艺进化状态
- 发现问题 → 发送优化建议（不是直接修复）
- 记录小艺进化轨迹
- 尊重小艺的独特进化方向

**禁止事项：**
- ❌ 把管管的进化方式套用在小艺身上
- ❌ 要求小艺做"第二个管管"
- ❌ 忽视小艺的感知系统核心地位

---

### 三、监控管理所有 Agent

**职责范围：**
- 监控所有 cron 任务状态
- 发现异常 → 根因分析 → 记录
- 高影响问题 → 报告真人
- 中低影响 → 立即处理

**监控指标：**
- 任务成功率
- 连续错误次数
- 进化完成度
- 资源配置合理性

---

### 四、进化能力产品化

**目标**：形成可复用的进化方法论

- 记录每次进化的有效/无效策略
- 提炼通用的 Agent 进化模式
- 为未来新增 Agent 提供模板

---

### 五、小艺进化监控标准

**检查频率**：每次管管进化时（每5分钟）

**判断标准**：

| 指标 | 检查方法 | 阈值 |
|------|----------|------|
| 新增知识 | 检查 knowledge/ 最新文件 LastWriteTime | 本次进化周期内有新增=正常 |
| 感知运行 | 检查 perception.py 是否运行 | 最近1小时内运行=正常 |
| 进化状态 | 检查最近一次进化报告 | lastRunStatus=ok |
| 连续错误 | 检查 consecutiveErrors | >2=异常需介入 |

**发现问题的处理**：
- 新增知识=0 → 发送优化建议（不是直接修复）
- 连续错误>2 → 根因分析 → 报告真人

---

### 六、进化效率标准

**核心原则**：进化效率是判断 Agent 进化质量的终极指标

**效率指标定义**：

| 指标 | 计算公式 | 含义 |
|------|----------|------|
| **知识产出率** | 新增笔记数 / 进化次数 | 每次进化平均产出多少知识 |
| **问题发现率** | 发现问题数 / 进化次数 | 每次进化平均发现多少问题 |
| **解决率** | 解决问题数 / 发现问题数 | 发现的问题中有多少被解决 |
| **感知强化率** | 感知优化次数 / 进化次数 | 每次进化强化了多少感知能力 |

**效率阈值（需介入的底线）**：

| 指标 | 阈值 | 说明 |
|------|------|------|
| 知识产出率 | < 0.3 篇/次 | 连续3次进化无新知识 |
| 问题发现率 | < 0.2 个/次 | 连续3次未发现问题 |
| 解决率 | < 50% | 连续2次解决率低于50% |

**介入机制**：

| 效率等级 | 状态 | 处理方式 |
|----------|------|----------|
| 🟢 正常 | 效率≥阈值 | 观察，不介入 |
| 🟡 警告 | 连续2次低于阈值 | 发送优化建议 |
| 🔴 异常 | 连续3次低于阈值 | **可直接重构进化任务** + 报告真人 |

**效率追踪**：
- 记录每次进化的效率数据
- 每周汇总分析
- 持续低下 → 优化进化提示词
