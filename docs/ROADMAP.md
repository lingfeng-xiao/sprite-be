# 10阶段开发路线图 (ROADMAP)

| 阶段 | 名称 | 目标 | 交付物 | 优先级 |
|------|------|------|--------|--------|
| S1 | 可靠性加固 | 解决P0阻断性问题 | 内存告警 + LLM降级 | P0 |
| S2 | 主人反馈学习 | P1-1 | 响应追踪 + 学习机制 | P1 |
| S3 | 情绪时间模式 | P1-2 | 时间维度情绪分析 | P1 |
| S4 | 记忆GitHub持久化 | P1-3 | 定时备份到GitHub | P1 |
| S5 | 传感器系统加固 | P1-4 | 真实数据 + 健康检查 | 技术债 |
| S6 | 决策引擎增强 | 增强决策能力 | 多维度决策 + 置信度 | 优化 |
| S7 | 动作系统扩展 | 扩展能力边界 | 新动作插件 | 优化 |

## 阶段1: 可靠性加固 (S1)

**目标**: 解决阻断性问题，确保系统稳定运行

### 任务
| ID | 任务 | 验收标准 | 涉及文件 |
|----|------|----------|----------|
| S1-1 | 服务器内存告警监控 | 内存>80%触发告警通知 | ProactiveService, RealPlatformSensor |
| S1-2 | LLM API失败降级处理 | API失败时自动切换到启发式推理 | MinMaxLlmReasoner, ReasoningEngine |
| S1-3 | 系统健康状态API | 提供/sprite/health端点 | SpriteController |

### 交付物
- 内存告警机制
- LLM降级方案
- 健康检查API
